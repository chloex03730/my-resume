# Chiikawa Interactive Resume

## Project Overview

This project is an interactive online resume inspired by **Chiikawa**, a popular Japanese character created by Nagano. The website presents Chiikawa's adventures, skills, education, and projects in the style of a personal resume.

The project was developed across six Front-End Web Technologies labs. Starting from a simple semantic HTML page, it was gradually enhanced with responsive layouts, CSS animations, and JavaScript features to create a modern and interactive website.

## Features

### Lab 1 – Semantic HTML
- Built a semantic resume using HTML5 elements.
- Structured the page with header, navigation, sections, articles, and footer.

### Lab 2 – CSS Styling
- Added a custom Chiikawa-inspired design.
- Used CSS variables, Flexbox, and Grid for layout.
- Improved typography with Google Fonts.

### Lab 3 – Responsive Design
- Applied a mobile-first approach.
- Added responsive layouts for mobile, tablet, and desktop.
- Used responsive images and fluid typography.

### Lab 4 – CSS Animations
- Added hover effects for navigation and cards.
- Created animated skill bars.
- Added page animations with accessibility support using prefers-reduced-motion.

### Lab 5 – JavaScript Interactivity
- Implemented dark mode.
- Added an accordion for the work experience section.
- Built a tabbed skills interface.
- Added a print button.

### Lab 6 – Dynamic Data & Deployment
- Implemented scroll reveal animations using the Intersection Observer API.
- Generated the skills section dynamically from a JavaScript data array while keeping animated skill bars.
- Built a contact form with client-side validation.
- Validated the project using the W3C HTML and CSS validators.

## Technologies Used

- HTML5
- CSS3
- JavaScript (ES6)
- CSS Grid
- CSS Flexbox
- Google Fonts
- Intersection Observer API

## Validation

- HTML validated using the W3C HTML Validator.
- CSS validated using the W3C CSS Validator.
- Tested on different screen sizes to ensure responsive behaviour.
